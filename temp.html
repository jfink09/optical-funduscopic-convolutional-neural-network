Diagnose Fundus Images with PyTorch Computer Vision

About The Model

User uploaded funduscopic images run through a convolutional neural network (CNN) which is an algorithm that recognizes patterns in image data, learns about these patterns, and predicts what the image is supposed to be. The CNN I used is the Tiny VGG architecture which consists of an input layer, convolutional layers that contain the learned kernels (weights) that looks for features in images that are different from one another. A convolutional neuron performs elementwise dot product operations with a unique kernel and the output of the previous layer's corresponding neuron. TinyVGG also containes the ReLU activation function to apply non-linearity to increase the accuracy of the model. There are max-pooling layers that use a 2x2 kernel and a stride of 2 to discard activations that make Tiny VGG computationally efficient and avoids overfitting. Finally, a flatten layer is included to conver 3-D layers into 1-D vectors.

Data Augmentation or Transfer Learning

After evaluation of the patient scans, we can use their scans to create accurate mesh models of the vessel and its pathology in CFD software to run fluid flow simulations. Useful data we can collect from CFD inclues wall shear stresses, pressure gradients, and other flow parameters. The goal will be attempting to optimze the CFD simulations through deep learning.
